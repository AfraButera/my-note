## `list(input())` vs `input().split()`

**Q：** 用 `list(input())` 輸入 `15 20 30` 時，為什麼程式把空格也當成元素？

**A：** `list()` 作用在字串時，會把字串的**每一個字元**逐一拆開，包括空格：

```python
list("15 20 30")
# → ['1', '5', ' ', '2', '0', ' ', '3', '0']
```

要按照空白分割成完整的數字，應該用 `.split()`：

```python
"15 20 30".split()
# → ['15', '20', '30']
```

`.split()` 的邏輯是「遇到空白就切一刀」，所以 15、100 這種多位數都能正確保留。

## `.split()` 是字串的方法，list 沒有

**Q：** 我試著寫 `list.split()`，發現不可行。所以 list 不能用 split 嗎？

**A：** 對，因為 `.split()` 是**字串（str）的方法**，不是 list 的方法。每種資料型態有自己的工具，方法只屬於特定類型：

```python
# ✅ 字串有 .split()
"15 20 30".split()     # → ['15', '20', '30']
input().split()        # input() 回傳字串，可以接 .split()

# ❌ list 沒有 .split()
[15, 20, 30].split()   # 報錯：list 沒有這個方法
```

比喻：`.split()` 是廚房刀，只能對食材（字串）用。對盤子（list）說「幫我切菜」，刀不認識盤子。

另外，`.split()` 回傳的結果**本身就已經是 list**，不需要再包 `list()`：

```python
input().split()         # 已經是 list
list(input().split())   # 多此一舉，但不會報錯
```

## 字串排序 vs 數字排序

**Q：** 我在把字串轉成 int 之前就做了 `sort()`，為什麼 `'9'` 跑到最後面去了？

**A：** 字串排序用的是**字典序**（逐字元比較 ASCII 碼），不是數值大小：

```python
# 字串排序（錯誤）
['9', '15', '20'].sort()
# → ['15', '20', '9']   ← '1' < '2' < '9'，所以 '9' 排最後

# 數字排序（正確）
[9, 15, 20].sort()
# → [9, 15, 20]         ← 數值比較，結果正確
```

字典序就像查字典——`'100'` 排在 `'2'` 前面，因為第一個字母 `'1'` < `'2'`，不管後面有幾位數。

**修正方式：** 先轉 int，再 sort：

```python
users = input().split()      # ['9', '15', '20']
users = list(map(int, users)) # [9, 15, 20]
users.sort()                  # [9, 15, 20] ← 正確數值排序
```

## `for` 迴圈手動轉型 vs `map` vs list comprehension

**Q：** 如果不用 `map`，最原始的方式要怎麼把 list 裡的字串逐一轉成數字？

**A：** 三種方式，做的事情完全一樣，只是抽象程度不同：

**方式一：手動 for 迴圈（最原始）**
```python
users_str = ['10', '20', '30']
users_int = []
for x in users_str:
    users_int.append(int(x))
# → [10, 20, 30]
```

逐圈追蹤：

| 圈數 | x | int(x) | users_int |
|------|---|--------|-----------|
| 1 | `'10'` | `10` | `[10]` |
| 2 | `'20'` | `20` | `[10, 20]` |
| 3 | `'30'` | `30` | `[10, 20, 30]` |

**方式二：list comprehension（縮寫版）**
```python
users_int = [int(x) for x in users_str]
```
意思和上面完全一樣，把「建空 list、迴圈、append」合成一行。

**方式三：map（更進一步）**
```python
users_int = list(map(int, users_str))
```
`map` 回傳的是「延遲執行的物件」，不是 list，所以要加 `list()` 才能看到結果、用 `len()` 算長度。
