## 🐍 Python｜作答區的起始變數，是否要重新宣告？

**Q：** 題目說「重複定義兩次」，是我沒有刪掉你寫的部分嗎？

**A：**
對。作答區的結構通常長這樣：

```python
letters = ['a', 'b', 'a', 'c', 'b', 'a']
count = {}
# 在這裡寫你的答案
```

前兩行是題目提供的起始變數，已經宣告好了。  
如果在 `# 在這裡寫你的答案` 下方又寫了一次：

```python
letters = ['a', 'b', 'a', 'c', 'b', 'a']   # ← 題目給的
count = {}                                    # ← 題目給的
# 在這裡寫你的答案
letters = ['a', 'b', 'a', 'c', 'b', 'a']   # ← 重複了
count = {}                                    # ← 重複了
for letter in letters:
    ...
```

就等於宣告了兩次，雖然不影響執行，但是多餘的。  
正確做法是直接在 `# 在這裡寫你的答案` 後面接邏輯，不重新宣告。

## 🐍 Python｜`print(dict)` 和 for 迴圈逐行印出，差異在哪裡？

**Q：** 為什麼 `print(count)` 不行？差異是在哪裡？

**A：**
`print(count)` 是把 dict「整個」印出來，Python 用它自己的格式輸出：

```python
print(count)
# {'h': 1, 'e': 1, 'l': 3, 'o': 2, 'w': 1, 'r': 1, 'd': 1}
```

Python 自動加上 `{}`、引號、逗號，這是 Python 內部的表示方式，不是給人看的排版。

for 迴圈讓你自己拿到每一筆 key 和 value，可以自由決定格式：

```python
for key, value in count.items():
    print(f"{key}: {value}")
# h: 1
# e: 1
# l: 3
# ...
```

一句話總結：`print(dict)` 是印「整個容器的樣子」，for 迴圈是印「每個內容物」，可以自訂格式。

## 🇬🇧 英文｜為什麼逐字查字效率低

**Q：** 查每個不懂的字，結果一大堆根本記不住，這樣學有意義嗎？

**A：**
人類單次接觸一個字，記住的機率極低，通常需要在不同情境看到 7–12 次才會自然記住。逐字查字除了耗時，也打斷理解節奏。

更有效的做法是只查「少了這個字這句話就完全看不懂」的關鍵字，其他跳過，讓大腦靠上下文猜測——這本身就是語言習得的一部分。

## 🇬🇧 英文｜可理解輸入（i+1）是什麼

**Q：** 推薦的英文學習核心原則是什麼？

**A：**
來自語言學家 Krashen 的「可理解輸入」理論：學習材料只比目前程度稍難一點點（i+1），才能有效吸收。太難會挫敗，太簡單沒進步。

初學者直接聽難度高的內容，30–40% 不懂是正常的，但超過這個比例就代表素材超出 i+1，應該換難度更低的教材先建立信心。

## 🇬🇧 英文｜技術文件英文比日常對話簡單

**Q：** 看技術文件需要特別訓練嗎？

**A：**
技術文件的英文反而比日常對話容易入門，原因有兩個：

1. **句型固定**：specify、configure、define 這類動詞高頻重複，熟悉後理解速度快
2. **已懂技術內容**：可以從上下文猜出不熟悉的字義，不需要每個字都查

最有效的練習就是直接讀自己熟悉領域的文件，遇到重複出現的英文詞記下來即可。

## 🇬🇧 英文｜被動輸入 vs 主動學習的差異

**Q：** 看脫口秀、英文學習 Reels 算有效嗎？

**A：**
效果差異在主動程度：

| 來源 | 類型 | 效果 |
|------|------|------|
| 脫口秀 | 被動輸入 | 中（長期有幫助，但進步慢） |
| 英文學習 Reels | 碎片知識 | 低（孤立知識點，容易忘） |
| MJ English + 逐字稿 | 主動學習 | 高 |

真正有效的學習需要主動處理：對照逐字稿、重複聽同一集、嘗試造句或跟讀。被動輸入可以當補充，不能當主力。

## ⚙️ 工具｜Docusaurus sidebar 自動生成 vs navbar 手動維護

**Q：** 每次在網站新增東西都會去對齊 navbar 跟 sidebar 嗎？

**A：**
兩個機制完全不同：

**Sidebar（左側目錄）— 自動生成**
`sidebars.js` 設定 `autogenerated`，只要：
- 資料夾有 `_category_.json`（設定標籤和排序位置）
- MD 檔有 `sidebar_position`（設定在分類裡的順序）

新增檔案就自動出現在 sidebar，不需手動維護。

**Navbar（頂部導覽列）— 手動維護**
每次新增內容都要去 `docusaurus.config.js` 的對應 dropdown 加一行，這個步驟容易漏掉。各提示詞（engup、readnote、pyup、ccq）都內建這個步驟，但直接新增筆記時需要特別注意。

## ⚙️ 工具｜Cron 是什麼

**Q：** Cron 是什麼？像 Windows 的 Task Scheduler 嗎？

**A：**
對，就是 Mac/Linux 版的工作排程器。讓你指定「什麼時間，執行什麼指令」，時間到自動跑，不需要人在場。

格式是 5 個欄位：`分 時 日 月 星期`，`*` 代表不限。

| 範例 | 意思 |
|------|------|
| `0 9 * * 3` | 每週三早上 9:00 |
| `0 9 * * 6` | 每週六早上 9:00 |
| `0 8 * * 1-5` | 週一到週五早上 8:00 |

搭配一支 Python 腳本：「時間到就執行這支腳本」，就能實現任何自動化。

## ⚙️ 工具｜Crontab 設定存在哪裡

**Q：** Cron 的設定存在哪裡？腳本又存在哪裡？

**A：**
兩個東西分開：

**Python 腳本** — 就是一般檔案，放在你指定的路徑，例如：
```
/Users/afralin/Desktop/ClaudeCode/telegram_bot/engwrite_reminder.py
```

**Crontab 設定** — 由系統管理，實際位置是：
```
/var/spool/cron/crontabs/afralin
```
但不直接編輯那個路徑，改用指令操作：

```bash
crontab -l    # 查看目前所有排程
crontab -e    # 編輯排程
crontab -r    # 刪除全部排程（小心）
```

類比 Windows：設定存在系統登錄裡，但你不去改 regedit，而是開 Task Scheduler GUI——`crontab` 指令就是那個 GUI 的角色。
