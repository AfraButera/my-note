## 一個 App 可以同時處理 IG 和 Facebook 發文嗎？

**Q：** 我想要 IG 發文時 Facebook 也同步發，我應該建兩個 App 嗎？

**A：** 不需要，一個 App 就夠。

Meta 的 App 可以同時加入多個使用案例，你只需要：
1. 加入「管理 Instagram 的訊息和內容」使用案例 → 取得 IG 發文權限
2. 加入「管理粉絲專頁的所有內容」使用案例 → 取得 Facebook 粉專發文權限

兩個使用案例共用同一個 App、同一組 App ID 和 App Secret，產出的 Token 也可以同時覆蓋兩邊。

拆成兩個 App 反而麻煩——要管兩組 Token、兩套憑證，沒有好處。

## Graph API Explorer 的權限下拉是空的，真正的原因是什麼？

**Q：** 之前說是 Safari 的問題，但後來發現不只這樣？

**A：** 是兩個原因同時發生，當時只找到一半：

1. **Safari JS 元件壞掉**：下拉選單完全無法渲染，什麼都不顯示
2. **App 使用案例未配置對應權限**：即使下拉正常，那些權限本來就不在清單裡

當時換到 Chrome 之後下拉正常了，就以為問題解決了。但實際上，同時也切換了 App 的使用案例（從 Instagram Login 換到含有 Facebook 登入的設定），兩個動作同步發生，才誤以為只是瀏覽器問題。

**結論**：Graph API Explorer 的下拉只會列出 App 已啟用的使用案例裡有的權限。沒有配置的權限，不管用什麼瀏覽器都不會出現。

## pages_manage_posts 是什麼？為什麼需要它？

**Q：** 這個權限是幹嘛的？

**A：** `pages_manage_posts` 是 Facebook Pages API 的權限，允許應用程式在指定的粉絲專頁上**建立、編輯和刪除貼文**。

沒有這個權限，你的 Token 只能「看」粉專的資料，無法代替粉專發文。就像你有一把可以進圖書館的鑰匙，但沒有館員的章，你能看書但不能上架新書。

加入「管理粉絲專頁的所有內容」使用案例後，這個權限才會出現在可申請的清單裡。

## 廣告帳號被限制，會影響 API 發文嗎？

**Q：** 我的粉絲專頁顯示「廣告帳號被限制」，我的 API 發文還能用嗎？

**A：** 不影響，這是兩件完全獨立的事。

「廣告帳號限制」是 Meta 對**付費投廣告**功能的限制，意思是你無法花錢在 Facebook 上推廣貼文或跑廣告。

但你用 API 發的是**一般有機貼文（organic post）**，不涉及廣告系統，完全不受這個限制影響。貼文照常發、照常顯示，兩件事互不干擾。

## Facebook 發文後在公開頁面看不到，是發文失敗嗎？

**Q：** API 回傳成功，但去粉絲專頁的公開頁面找不到貼文？

**A：** 不是失敗，是找的地方不對。

有時候剛透過 API 發出的貼文，公開頁面（一般訪客看到的樣子）需要幾秒鐘才會更新。正確的確認方式是：

**Meta Business Suite → 內容 → 貼文**

這裡是管理後台，貼文發出後幾乎即時可見，比公開頁面早顯示。重新整理公開頁面後通常也會出現。
