## Sidebar 和 Navbar Dropdown 有什麼差別？

**Q：** 我的 homepage sidebar 是看不出資料夾分類的對嗎？他是用 `[xx|]` 區分嗎？

**A：** 分兩個地方要分開理解：

**Sidebar（左側欄，進入文章後才出現）**
由資料夾結構自動產生，`CodeShiba學習練習/`、`練習筆記/`、`Claude教學練習紀錄/` 各自是一個可折疊的分類，看得出層級。由 `_category_.json` 控制名稱與順序。

**Navbar Dropdown（頂部選單點開的清單）**
是手動寫在 `docusaurus.config.js` 的平面清單，沒有分組功能（beta.6 限制），所以用 `CodeShiba｜`、`練習｜`、`Claude教學｜` 前綴來視覺上區分。

從首頁點進去某一篇文章後，左側 sidebar 才會出現，那裡的分類是看得清楚的。Navbar 的平面清單只是入口捷徑。

## Docusaurus beta.6 是什麼？

**Q：** beta.6 是什麼？

**A：** 就是 Docusaurus 的版本號。

`beta.6` = **2.0.0-beta.6**，是 2021 年左右的測試版。目前最新是 **3.x 正式版**，差了很多個版本。

你的網站當初建的時候用的就是這個版本，所以有些新功能在這個舊版本裡沒有，用了就會報錯。

## type: 'html' 是什麼？為什麼會報錯？

**Q：** `type: 'html'` 是幹嘛的？為什麼在這個版本用了會報錯？

**A：** 在 Docusaurus 新版裡，`type: 'html'` 讓你在 navbar dropdown 裡插入**任意 HTML**，用來做分隔線或小標題，達到分組效果：

```
資料型別
────────────    ← <hr/> 分隔線
CodeShiba       ← <small> 小標題
  if 判斷式
  數學運算
```

但因為這個功能是**後來才加進去**的，beta.6 的時候還不存在。Docusaurus 讀到 `type: 'html'`，查自己支援的類型清單，找不到就報錯。

類比：叫一支 iOS 12 的手機跑只有 iOS 16 才有的功能，手機不認識，直接閃退。

解法：改回用 label 前綴（`CodeShiba｜`、`練習｜`）來視覺區分，這在所有版本都支援。

## 問答內容會記錄在 dailylog 嗎？

**Q：** 你這些也會記錄在 dailylog 嗎？

**A：** 不會，dailylog 只記錄「做了什麼事」，像問答解說不算協作事項，不會寫進去。概念性問答改用 `ccq` 指令整理，存到網站的「Claude 問答」分類。

## YAML frontmatter 的 id 為什麼要加引號？

**知識點：** Docusaurus MD 檔的 frontmatter `id` 若是日期格式，不加引號會出錯。

**原因：** YAML 會把 `2026-05-06` 自動解讀為日期物件（Date），Docusaurus 再把它轉成字串時變成：

```
Wed May 06 2026 08:00:00 GMT+0800 (台北標準時間)
```

這個字串變成 URL 路徑就完全爛掉，導致 broken links 報錯。

**正確寫法：加引號強制當字串**

```yaml
---
id: "2026-05-06"   ✅
title: 問答紀錄 2026-05-06
---
```

## Docusaurus 資料夾名稱為什麼不能用中文？

**知識點：** `docs/` 下的資料夾名稱若含中文，會導致 URL 不一致。

**原因：** 中文字元在 URL 裡需要 percent-encoding（如 `問` → `%E5%95%8F`）。Docusaurus 產生的實際頁面路徑是編碼後的 URL，但 navbar 的 `to` 若寫中文，兩者對不上，就出現 broken links。

**解法：** 資料夾名稱一律用英文，例如 `ClaudeQA/` 而不是 `Claude問答/`。
