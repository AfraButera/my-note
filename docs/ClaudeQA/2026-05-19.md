## 🐍 Python｜`*args` 有沒有，跟「能接幾個參數」有什麼關係？

**Q：** 你說沒有 `*args` 的話只能接一個參數，但像 `greet(name, greeting)` 或 `min_max(nums)` 這些不都有兩個、一個參數嗎？

**A：** 說法有誤，應該更正：

- 沒有 `*` → 數量**固定**，你定義幾個參數，呼叫者就只能傳幾個，多傳少傳都會報錯
- 有 `*` → 數量**不限**，呼叫者傳幾個都接得住

`*args` 解決的不是「能不能傳多個」，而是「**傳入的數量在寫程式時還不知道**」的情境：

```python
# 固定兩個參數：你知道一定會有 name 和 greeting
def greet(name, greeting='你好'):
    ...

# *args：你不知道呼叫者會傳幾個數字進來
def total(*nums):
    return sum(nums)

total(1, 2, 3)          # ✅
total(1, 2, 3, 4, 5)    # ✅
```

## 🐍 Python｜`*args` 的真正作用是什麼？

**Q：** `*args` 的作用到底是？只是防止人家多輸入嗎？

**A：** 不是「防止」，而是**讓呼叫者可以傳多個獨立的值，不用先包成 list**。

關鍵是區分「輸入」和「輸出」是兩件事：

| 概念 | 方向 | 意思 |
|------|------|------|
| `return a, b, c` | 輸出 | 你（寫函式的人）決定回傳幾個值，寫死的 |
| `*args` | 輸入 | 呼叫者決定傳幾個值進來，每次可以不同 |

```python
# 沒有 *args → 呼叫者必須傳一個 list
def total(nums):
    return sum(nums)

total([1, 2, 3])    # ✅
total(1, 2, 3)      # ❌ 報錯

# 有 *args → 呼叫者直接傳個別的值
def total(*nums):
    return sum(nums)

total(1, 2, 3)      # ✅ nums = (1, 2, 3)
```

兩個版本算出一樣的結果，差的是**呼叫方式**。

## 🐍 Python｜`return` 可以回傳超過兩個值嗎？

**Q：** `return min(nums), max(nums)` 這樣，我可以再加 `sum(nums)` 變成三個嗎？會報錯嗎？

**A：** 不會報錯，完全合法。`return` 後面可以放任意數量的值，Python 全部自動包成一個 tuple：

```python
def min_max_sum(nums):
    return min(nums), max(nums), sum(nums)
# 回傳 (1, 7, 13)

small, large, total = min_max_sum([3, 1, 7, 2])
# small=1, large=7, total=13
```

接收時對應拆開就好，幾個都行。

## 🐍 Python｜`kwargs` 的 `kw` 是什麼的縮寫？

**Q：** `kwargs` 的 `kw` 是知識（knowledge）的意思嗎？

**A：** 不是，`kw` 是 **keyword**（關鍵字）的縮寫：

`kwargs` = **k**ey**w**ord **arg**ument**s** → 關鍵字引數

對比兩種傳參方式：

```python
# positional arguments（位置引數）：不帶名字，靠位置區分
total(1, 2, 3)

# keyword arguments（關鍵字引數）：帶名字的 key=value 形式
greet(name='Ziv', city='台灣')
```

`**kwargs` 收的就是帶名字那種，全部包成 dict。
