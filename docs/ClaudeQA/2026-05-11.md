## `for` 和 `while True` 各自適合什麼情況？

**Q：** 老師（Gemini）叫我用 `while`，你說用 `for`，到底誰對？

**A：** 兩個都能跑，但使用情境不同：

- **`for` 迴圈**：知道要跑幾次，或是在遍歷一個序列時用。例如印出清單裡的每個元素。
- **`while True`**：「不知道要跑幾次，直到某個條件才停」時用。例如一直等使用者輸入，直到選了「離開」。

這題的選單需要「持續等待使用者選擇，直到選離開才結束」——這是 `while True` 的典型場景。`enumerate` 是用來印選單的工具，不是控制「要不要繼續」的工具。

## `break` 要放在哪種迴圈裡才有意義？

**Q：** `break` 放在 `for` 裡和放在 `while True` 裡有什麼不同？

**A：** `break` 只能跳出「它所在的那一層迴圈」。

如果 `break` 放在 `while True` 裡：
- 使用者選了「離開」→ `break` → 整個 `while True` 結束 ✓

如果 `break` 放在 `for` 迴圈裡（沒有外層 while）：
- `for` 跑完一次清單就結束了，`break` 只是提前跳出這一次遍歷，程式照樣結束
- 使用者根本沒辦法「再選一次」

所以 `break` 要發揮「程式一直跑直到你說停」的效果，就必須在 `while True` 裡。

## for 迴圈的變數在迴圈結束後還存在嗎？

**Q：** 我用 `print(f"{option}")` 印離開，`option` 是從哪來的？

**A：** Python 的 `for` 迴圈變數在迴圈結束後不會消失，它會保留最後一次的值。

```python
for i, opt in enumerate(['A', 'B', 'C']):
    pass

print(opt)  # 印出 'C'（最後一次的值）
```

所以在選單裡，`for` 跑完後 `option` 的值是最後一個元素「離開」。`print(f"{option}")` 剛好印出「離開」是因為湊巧，不是故意設計的。更清楚的寫法是直接寫 `print("離開")` 或 `print(options[users-1])`，讓意圖明確。

## 把第一個 for 移進 while True 後，為什麼第二個 for 就多餘了？

**Q：** 我把印選單的 for 放進 while True 裡面，但又在 elif 裡放了另一個 for，結果選完之後選單印了兩次？

**A：** 因為 `while True` 每跑一圈都會從頭開始執行。

原本的問題：選單只印一次（for 在 while 外面）→ 解法是把 for 放進 while True。

放進去之後的流程：
```
while True 開始
  → for 印選單       ← 第一次印
  → 使用者選 2
  → elif 印「你選了鳳梨」
  → elif 裡的 for 又印選單  ← 多餘！
while True 再一圈
  → for 印選單       ← 這裡本來就會再印
```

只需要在 `while True` 頂端印一次就夠了，下一圈自然會再印，不需要在 `elif` 裡再印一次。
