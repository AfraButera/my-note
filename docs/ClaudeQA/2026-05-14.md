## 🐍 Python｜list comprehension 裡 if-else 放哪裡？

**Q：** 在 list comprehension 裡想用 if-else 標記每個元素，但一直語法錯誤，`if` 到底該放哪？

**A：** list comprehension 的 `if` 有兩個位置，行為完全不同：

```python
# 位置一：for 後面 → 篩選（filter）
# 不符合的元素直接被丟掉，不能接 else
[x for x in arr1 if x >= 60]
# → 只留下 >= 60 的元素

# 位置二：for 前面 → 對每個元素決定放什麼值
# 每個元素都進來，由 if-else 決定輸出什麼
['pass' if x >= 60 else 'fail' for x in arr1]
# → 每個元素都被標記，長度不變
```

**標記題**（每個元素都要有結果）用位置二；**篩選題**（只要符合條件的）用位置一。

## 🐍 Python｜print() 為什麼不能放進 list comprehension 當值？

**Q：** 我試了 `[print('pass') if x >= 60 else print('fail') for x in arr1]`，格式對了但 list 裡全是 None，為什麼？

**A：** `print()` 做兩件事：
1. 把文字**顯示在螢幕上**（副作用）
2. 執行完後**給出一個值**，那個值永遠是 `None`

list comprehension 收集的是每個表達式**給出的值**，不是螢幕上印的東西：

```python
# print('pass') 給出 None → arr2 收到 None
arr2 = [print('pass') if x >= 60 else print('fail') for x in arr1]
# arr2 = [None, None, None, None, None, None]  ← 文字印出來了，但 list 全空

# 直接放字串 → arr2 收到 'pass'
arr2 = ['pass' if x >= 60 else 'fail' for x in arr1]
# arr2 = ['pass', 'pass', 'fail', 'pass', 'pass', 'pass']  ✅
```

`print()` 是「做一件事」的動作，不是「給出一個值」的表達式。要放進 list，必須用**值**，不能用**動作**。

## ⚙️ 工具｜Docusaurus sidebar 的順序怎麼控制？

**Q：** 網站左側 sidebar 的目錄順序跟 navbar 不一樣，要怎麼讓它們一致？

**A：** Docusaurus 用 `autogenerated` sidebar 時，排序規則是：
- 有 `_category_.json` 且設了 `position` → 用那個數字排序
- 沒有 `_category_.json` → 按字母順序

所以只要在每個頂層資料夾裡放一個 `_category_.json`，設好 `position`，就能控制順序：

```json
{
  "label": "Docker",
  "position": 1
}
```

數字小的排前面。只要讓各資料夾的 `position` 跟 navbar 的左右順序對齊，兩邊就會一致。

## ⚙️ 工具｜ClaudeQA sidebar 排序為什麼亂掉？

**Q：** 問答紀錄在 homepage 按時間排，但 sidebar 卻亂掉，為什麼？

**A：** 原因是 `sidebar_position` 重複了 — `2026-05-12.md` 和 `2026-05-14.md` 都設了 `sidebar_position: 4`，Docusaurus 遇到重複數字時行為不可預測，所以排序亂掉。

根本解法是**移除所有 `sidebar_position`**。沒有這個欄位時，Docusaurus 會按**檔名字母順序**排列。因為檔名格式是 `YYYY-MM-DD`，字母順序 = 日期順序，永遠自動時間排序，不需要手動維護。

```
2026-05-06.md → 第 1 位（字母序最小）
2026-05-07.md → 第 2 位
2026-05-11.md → 第 3 位
...
```

之後新增的 ccq 檔案也不需要帶 `sidebar_position`。
